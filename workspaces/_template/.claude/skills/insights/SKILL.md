# Insights Skill — 工作感悟记录

本 skill 规范工作中碎片化感悟的**快速记录、分类整理和回顾**。

适合捕捉那些稍纵即逝的想法：一句话的体会、踩坑后的反思、对某个问题的新认知、值得沉淀的经验片段。

---

## 数据文件结构

```
memory/
└── insights/
    ├── index.md                  # 分类索引（每次新增后更新）
    └── YYYY-MM.md                # 按月归档，同月感悟写入同一文件
```

所有路径从 `SESSION_CONTEXT.md` 读取 `Memory dir`，拼接子路径，**不使用硬编码**。

---

## 触发识别

| 用户说 | 触发动作 |
|--------|---------|
| 「记录一下」「有个感悟」「想法」「体会」 | → **快速记录** |
| 「看看我的感悟」「回顾一下」「之前记过什么」 | → **回顾** |
| 「关于 XX 有没有感悟」「搜一下」 | → **检索** |

---

## 功能一：快速记录

### 操作流程

1. 用户说出感悟内容
2. AI 提炼一个简短标签（2-4 字，作为标题）
3. 确认分类（见下方分类表），若不确定直接默认「综合」
4. 写入当月文件，更新索引

### 分类参考

| 分类 | 适用内容 |
|------|---------|
| 管理 | 带团队、沟通协作、人员问题 |
| 测试 | 质量、流程、方法论、工具 |
| 成长 | 个人认知、学习方式、职业思考 |
| 项目 | 需求、排期、风险、复盘 |
| 综合 | 不好归类的零散想法 |

分类由 AI 自动判断，用户可纠正。

### 感悟条目格式

```markdown
### [简短标题]

> **[分类]** · YYYY-MM-DD

[感悟正文，保留用户原始表达，AI 可适当润色但不改变意思]

---
```

### 写入流程

```bash
# 1. 确定当月文件路径
MONTH_FILE="<memory_dir>/insights/$(date +%Y-%m).md"

# 2. 若文件不存在则初始化
flock -x <memory_lock_path> -c "
  mkdir -p <memory_dir>/insights
  if [ ! -f ${MONTH_FILE} ]; then
    cat > ${MONTH_FILE} << 'EOF'
# 工作感悟 — YYYY年MM月

EOF
  fi
  # 3. 追加感悟条目
  cat >> ${MONTH_FILE} << 'EOF'
### [标题]

> **[分类]** · YYYY-MM-DD

[正文]

---
EOF
"

# 4. 更新索引（见功能三）
```

## 功能二：回顾

### 触发方式

| 用户说 | 读取范围 |
|--------|---------|
| 「看看最近的感悟」 | 当月文件 |
| 「上个月记了什么」 | 上月文件 |
| 「今年的感悟」 | 当年所有月份文件 |
| 「关于管理的感悟」 | 全部文件，过滤分类=管理 |

### 输出格式

```
💡 **工作感悟回顾**（2026年3月）

共 N 条

**测试**
- [2026-03-10] 标题：一句话摘要

**管理**
- [2026-03-05] 标题：一句话摘要

查看某条完整内容，说「展开第 N 条」
```

## 功能三：索引维护

每次新增感悟后，同步更新 `insights/index.md`。

### 索引文件格式

```markdown
# 感悟索引

> 最后更新：YYYY-MM-DD，共 N 条

## 管理
- [YYYY-MM-DD 标题](YYYY-MM.md) — 一句话摘要

## 测试
- ...

## 成长
- ...

## 项目
- ...

## 综合
- ...
```

### 写入规则

- 每条索引一行，按日期倒序排列（新的在上）
- 摘要控制在 20 字以内

---

## 功能四：检索

按关键词或分类搜索历史感悟。

```bash
# 按关键词搜索
grep -r "<关键词>" <memory_dir>/insights/ --include="*.md" -l

# 按分类过滤
grep -A 5 "\*\*管理\*\*" <memory_dir>/insights/*.md
```

输出匹配条目，附文件和日期信息。

---

## 注意事项

- 保留用户原始表达，AI 润色以不改变原意为原则
- 感悟条目写入后不得删除，只允许追加「后记」
- 索引文件每次新增后必须同步更新
- 涉及具体同事或敏感情境，建议用角色代称（如「某 leader」「开发侧」）
